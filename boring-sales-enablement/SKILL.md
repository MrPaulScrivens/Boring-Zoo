## Core Principles

### Authenticity Over Polish
Sales materials should sound like your Voice, not generic corporate. If it sounds like everyone else's pitch deck, it won't land differently.

### World Code Story Arc
Every piece of sales collateral follows the World Code arc: Before State → Method → Climax Transformation. This isn't a feature tour — it's a transformation story.

### Scannable Over Comprehensive
People need information fast. Use bold headers, short bullets, and visual hierarchy.

### Tie Back to Transformation
Every claim connects to the Climax promise. Features mean nothing without the "which means [transformation outcome]."

## Sales Deck / Pitch Deck — World Code Structure

### Slide Framework

1. **The Before State** — The pain your buyer lives with (from `climax.md`)
2. **Cost of Staying** — What the Before State costs them
3. **The Wrong Belief** — Why conventional approaches fail (from `conversation.md`)
4. **Your World Code** — The shift in thinking (Core Message from `conversation.md`)
5. **The Method** — How your approach works differently (from `method.md`)
6. **Method Phases** — Walk through the key phases (from `method.md`)
7. **The Climax** — Transformation promise with proof (from `climax.md`)
8. **Proof Points** — Results, testimonials, case studies
9. **The Creation** — What you're offering (from `creation.md`, BONES)
10. **Investment** — Pricing from `creation.md`
11. **The Crossing** — Next steps, how to get started (from `crossing.md`)

### Deck Principles

- **Story arc, not feature tour.** Follow the World Code narrative: Before State → Method → Transformation.
- **One idea per slide.** If you need two points, use two slides.
- **Voice throughout.** This deck should sound like the user, not a template.

## Objection Handling — World Code Style

### Primary Objection Source

The Real Objection from `crossing.md` is your starting point. Build outward from there:

### Response Framework

For each objection:
1. **Objection** — How people express it
2. **Real Concern** — What's actually behind it (reference Crossing insights)
3. **World Code Response** — Reframe using Method logic and Climax promise
4. **Proof** — Specific evidence
5. **Bridge Question** — Move conversation forward

### Common Objection Categories (World Code Framing)

| Category | World Code Response |
|----------|-------------------|
| "Too expensive" | Reference Creation BONES value, Climax transformation ROI |
| "Not the right time" | Before State is costing them NOW |
| "Already have something" | Method's "Why It Works Differently" |
| "Not sure it'll work for me" | Climax "Who This Is For" + proof |
| "Need to think about it" | Address the Real Objection from Crossing |

## Demo Scripts & Talk Tracks

### World Code Demo Structure

1. **Opening** (2 min) — Before State acknowledgment, set context in Voice
2. **Discovery recap** (3 min) — Confirm their Before State pain
3. **Method walkthrough** (15-20 min) — Show Method phases solving their pain
4. **Climax moment** (5 min) — Show the transformation outcome
5. **Close** (5 min) — Creation details, Crossing next steps

---

## Case Study Format — World Code Style

1. **Before State** — Where they were (maps to `climax.md`)
2. **Discovery** — How they found your Method
3. **Application** — Method phases they went through
4. **Transformation** — The Climax result with specific metrics
5. **In Their Words** — Quote (in their voice, not yours)
6. **Tags** — Audience segment, Method phase, use case
