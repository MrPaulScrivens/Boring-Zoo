# World Code → Skill Architecture Mapping — Reference

How each World Code element translates into skill components.

## Direct Mapping

### Voice → Skill Communication Style
**World Code role:** How you communicate. Your tone, rhythm, hard rules.
**Skill role:** How the skill speaks. Every output follows your voice.md rules.

**Implementation:**
- Read `world-code/voice.md` at skill start
- Apply tone and rhythm to all generated content
- Follow hard rules without exception
- Test outputs with: "Would I actually say this?"

### Climax → Skill Outcome / North Star
**World Code role:** The transformation you promise. The big result.
**Skill role:** What the skill points toward. The outcome it delivers or contributes to.

**Implementation:**
- Read `world-code/climax.md` to understand the transformation promise
- Every skill should connect to the Climax in some way
- The skill's output should move someone closer to the Climax
- Use Climax language in the skill's description and framing

### Code/Method → Process Steps
**World Code role:** Your unique methodology. How your Character achieves the Climax.
**Skill role:** The sequential steps the skill walks someone through.

**Implementation:**
- Read `world-code/method.md` for the methodology framework
- Each phase/step of the Method becomes a process step in the skill
- The skill follows the Method's logic and sequence
- Reference Method principles at each step

### Creation → The Skill Itself
**World Code role:** Your offer. The packaged thing people buy/use.
**Skill role:** The skill IS a Creation. It packages methodology into an interactive experience.

**Implementation:**
- Apply B.O.N.E.S. test to evaluate skill quality:
  - **B**ig: Solves a real problem
  - **O**bvious: Clear what they get
  - **N**ew: Interactive format feels fresh
  - **E**asy: Doable in one session
  - **S**afe: Low/no risk
- Skill inherits Creation philosophy: delivery matters as much as content

### Conversation → Discovery Questions
**World Code role:** Content themes. Beliefs you challenge. What you talk about.
**Skill role:** What the skill asks about. Discovery question design.

**Implementation:**
- Read `world-code/conversation.md` for themes and beliefs
- Content themes become question topics
- Wrong beliefs to challenge become diagnostic questions
- "What do you believe about X?" questions surface alignment/misalignment

### Crossing → Post-Skill Navigation
**World Code role:** How people enter and move deeper into your world.
**Skill role:** What happens after the skill finishes. Next steps. Connected skills.

**Implementation:**
- Read `world-code/crossing.md` for journey design
- End each skill with a clear next step
- Point to related skills when relevant
- Free skill → paid system mirrors Crossing entry point design

## The 80/20 Claim

If a creator has a complete World Code, 80% of skill-building is already done:

| Already Done (World Code) | Remaining (Skill-Specific) |
|---|---|
| Voice/tone rules | Question sequence order |
| Transformation defined | Output file format |
| Methodology mapped | Discovery question wording |
| Offer philosophy clear | Reference file selection |
| Content themes identified | Testing and refinement |
| Journey designed | Hub/connection architecture |

## For Creators Without a World Code

The Knowledge Anatomy (Chapter 3) works independently. The World Code mapping is an accelerator, not a requirement.

Without World Code, creators need to define:
- Voice: through the skill's tone instructions
- Outcome: through the skill's description and output
- Method: through the process steps directly
- Questions: from their coaching/teaching experience
- Navigation: by planning which skills connect

The World Code just means these things are already documented and ready to reference.
