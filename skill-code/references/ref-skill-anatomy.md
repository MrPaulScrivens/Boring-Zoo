# The Knowledge Anatomy — Reference

The 6-part framework for converting knowledge into interactive AI skills.

## The Six Parts

### 1. Prerequisites
What someone needs before starting. Not academic prerequisites — real-world readiness markers.
- Do they have the foundation this skill builds on?
- If not, where should they go first?
- Check for prerequisite files from other skills

### 2. Context
What the skill needs to know about THIS person. Factual information that shapes the experience.
- Business type, audience size, experience level
- What they've already tried
- Current state vs. desired state

### 3. Discovery
Questions that reveal where they actually are. Diagnostic, not factual.
- Surfaces the real issue, not the stated one
- Pushes vague answers toward specifics
- Reveals their language, beliefs, and blockers
- Usually 3-5 questions, asked one at a time

### 4. Process
The sequence of steps producing the transformation. This is the methodology.
- Each step builds on the previous
- Adapts based on discovery answers
- References the creator's unique framework
- Not a straight line — responds to the terrain

### 5. Synthesis
Taking all answers and producing something tailored.
- Combines context + discovery + process responses
- Not a template with blanks filled in
- Reflects their specific situation
- Applies the creator's voice and methodology

### 6. Output
The tangible deliverable. A file, plan, draft, or document.
- Specific file name and format
- Clear sections with defined content
- Presented for review before saving
- Allows 1-2 rounds of refinement

## Quality Check

A complete skill anatomy should:
- [ ] Have clear prerequisites (even if "none")
- [ ] Gather context before asking deeper questions
- [ ] Ask discovery questions one at a time
- [ ] Follow a defined process sequence
- [ ] Synthesize answers into something personalized
- [ ] Produce a tangible, named output file
